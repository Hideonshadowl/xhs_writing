# 更新双平台帖子数据

用浏览器打开创作者后台，读取已发内容的流量与互动，写回本仓库对应文件。默认执行完整流程，不要只汇报不落盘（除非用户明确说「只看不写」）。

## 触发词

更新数据 / 拉数据 / 同步数据 / 刷新数据 / 数据复盘 / 看流量 / 看互动 / 抓后台数据

## 后台入口

| 平台 | 列表页（优先） | 备用 |
| --- | --- | --- |
| 小红书 | https://creator.xiaohongshu.com/new/note-manager | 数据看板侧栏「数据看板」 |
| 抖音 | https://creator.douyin.com/creator-micro/content/manage | 数据中心 → 作品分析 / 账号总览 |

账号名参考：小红书「故事里见真章」。Cursor 内置浏览器与系统 Chrome **不共享登录态**。

## 执行流程

复制并跟踪：

```
进度：
- [ ] 1. 打开两边创作者后台并确认已登录
- [ ] 2. 抓取全部可见作品指标
- [ ] 3. 按标题匹配 posts/*.md
- [ ] 4. 回写发布状态 + 数据复盘
- [ ] 5. 同步选题池状态
- [ ] 6. 向用户汇报摘要表
```

### 1. 登录

1. 用 browser 打开上述两个 URL（可分标签）。
2. 若跳到扫码/手机号登录页：停止抓取，请用户在当前 Cursor 浏览器里扫码，等用户说「已登录」后再继续。
3. 已登录标志：小红书出现笔记列表或账号名；抖音出现「作品管理」列表。

### 2. 抓取字段

从**内容列表**读取每条作品（滚动直到「没有更多」）：

| 字段 | 小红书 | 抖音 |
| --- | --- | --- |
| 标题 | 笔记标题 | 作品标题 |
| 发布时间 | 列表时间 | 列表时间 |
| 阅读/播放 | 阅读（眼睛） | 播放 |
| 点赞 | 赞 | 赞 |
| 收藏 | 藏 | 藏 |
| 评论 | 评 | 评 |
| 分享 | 有则记 | 列表通常无 |

主页访问 / 新增关注：列表没有就写「后台列表未显示」，不要臆造。数据看板加载失败可跳过，以列表为准。

### 3. 匹配归档

1. 读 `posts/*.md` 的标题（`#` 标题或「标题最终版」）与发布状态。
2. 用标题关键词匹配后台作品；一篇可只在单平台有。
3. 匹配不上的作品：在汇报里单独列出，询问是否新建归档，**不要擅自新建**。

### 4. 回写 `posts/`

更新元信息：

```markdown
- 发布状态：已发布（小红书 + 抖音）  # 或「仅小红书」「仅抖音」
- 发布时间：小红书 YYYY-MM-DD HH:mm；抖音 YYYY-MM-DD HH:mm
```

单平台只写有数据的一侧。

更新「数据复盘」（保留私信/高赞评论/下一篇延展等人工字段，只改数据行）：

```markdown
## 数据复盘

- 数据拉取时间：YYYY-MM-DD
- 曝光量 / 阅读量：小红书阅读 N；抖音播放 N
- 点赞 / 收藏 / 评论：小红书 A / B / C（分享 D）；抖音 E / F / G
- 主页访问 / 新增关注：后台列表未显示
```

单平台示例：

- `小红书阅读 33；未发抖音`
- `未发小红书；抖音播放 0`
- 无分享则省略「（分享 D）」

### 5. 同步 `选题池.md`

对已匹配且已发的选题：状态改为 `已发布`；备注可补「双平台 / 仅小红书 / 仅抖音」及显著数据（如领先播放量）。未发布选题不动。

### 6. 汇报

用中文给一张对照表：标题 | 平台 | 阅读/播放 | 赞 | 藏 | 评 |（小红书分享）。点出当前表现最好的 1 篇。说明哪些只单平台、哪些未匹配。

## 约束

- 只改与本次数据相关的 `posts/*.md` 与 `选题池.md`；不顺手改正文、封面、规范文档。
- 不 commit、不 push，除非用户明确要求。
- 数字以当次后台页面为准；不确定就再打开列表核对，不估不编。
- 解锁 browser 前完成抓取；需要用户扫码时先 unlock。
