# 更新双平台帖子数据

用 **opencli**（Browser Bridge → 系统 Chrome 登录态）拉取已发内容的流量与互动，写回本仓库对应文件。默认执行完整流程，不要只汇报不落盘（除非用户明确说「只看不写」）。

**不要**用 Cursor 内置浏览器抓数。

## 触发词

更新数据 / 拉数据 / 同步数据 / 刷新数据 / 数据复盘 / 看流量 / 看互动 / 抓后台数据

## 前置条件

```bash
opencli doctor
```

需看到 Daemon + Extension 均为 OK。若 Extension 未连接：请用户确认 Chrome 已装 OpenCLI 扩展且已启用。若站点未登录：请用户在系统 Chrome 打开并登录创作者后台后再跑。

账号名参考：小红书「故事里见真章」。

## 执行流程

复制并跟踪：

```
进度：
- [ ] 1. opencli doctor 确认 Bridge 连通
- [ ] 2. 拉取小红书 + 抖音作品列表
- [ ] 3. 按标题匹配 posts/*.md
- [ ] 4. 回写发布状态 + 数据复盘
- [ ] 5. 同步选题池状态
- [ ] 6. 向用户汇报摘要表
```

### 1. 连通性

```bash
opencli doctor
```

失败则停止，按报错让用户处理扩展/登录，不要改用 Cursor browser。

### 2. 抓取

并行执行：

```bash
opencli xiaohongshu creator-notes --limit 20 -f yaml --window background
opencli douyin videos --limit 20 -f yaml --window background
```

| 字段 | 小红书 `creator-notes` | 抖音 `videos` |
| --- | --- | --- |
| 标题 | `title` | `title`（取首行/去标签后匹配） |
| 发布时间 | `date`（如 `2026年07月16日 16:34`） | `create_time`（如 `2026/7/16 18:01:29`） |
| 阅读/播放 | `views` | `play_count` |
| 点赞 | `likes` | `digg_count` |
| 收藏 | `collects` | 接口未返回 → 写「接口未返回」 |
| 评论 | `comments` | 接口未返回 → 写「接口未返回」 |
| 分享 | 无 | 无 |

主页访问 / 新增关注：写「后台列表未显示」，不要臆造。

时间格式写回时统一为 `YYYY-MM-DD HH:mm`（抖音去掉秒即可）。

### 3. 匹配归档

1. 读 `posts/*.md` 的标题（`#` 标题或「标题最终版」）与发布状态。
2. 用标题关键词匹配；一篇可只在单平台有。
3. 备选稿（B/C 版等）不要误标为已发布，除非标题与后台完全对上且用户确认。
4. 匹配不上的作品：在汇报里单独列出，询问是否新建归档，**不要擅自新建**。

### 4. 回写 `posts/`

更新元信息：

```markdown
- 发布状态：已发布（小红书 + 抖音）  # 或「仅小红书」「仅抖音」
- 发布时间：小红书 YYYY-MM-DD HH:mm；抖音 YYYY-MM-DD HH:mm
```

单平台只写有数据的一侧。

更新「数据复盘」（保留私信/高赞评论/下一篇延展等人工字段，只改数据行）：

```markdown
## 数据复盘

- 数据拉取时间：YYYY-MM-DD
- 曝光量 / 阅读量：小红书阅读 N；抖音播放 N
- 点赞 / 收藏 / 评论：小红书 A / B / C；抖音 E / 接口未返回 / 接口未返回
- 主页访问 / 新增关注：后台列表未显示
```

单平台示例：

- `小红书阅读 33；未发抖音`
- `未发小红书；抖音播放 1`
- 抖音有藏/评时再写成 `E / F / G`

### 5. 同步 `选题池.md`

对已匹配且已发的选题：状态改为 `已发布`；备注可补「双平台 / 仅小红书 / 仅抖音」及显著数据（如领先播放量）。未发布选题不动。

### 6. 汇报

用中文给一张对照表：标题 | 平台 | 阅读/播放 | 赞 | 藏 | 评。点出当前表现最好的 1 篇。说明哪些只单平台、哪些未匹配、抖音哪些字段接口未返回。

## 约束

- 只改与本次数据相关的 `posts/*.md` 与 `选题池.md`；不顺手改正文、封面、规范文档。
- 不 commit、不 push，除非用户明确要求。
- 数字以当次 opencli 输出为准；不确定就再跑一遍命令，不估不编。
- 禁止改用 Cursor 内置浏览器完成本 skill。
