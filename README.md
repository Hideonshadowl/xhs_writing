# 文稿服务双平台运营项目

本项目用于持续运营一个专业文稿内容账号，覆盖小红书与抖音。

主要引流受众：基层公务员、基层选调生、社区员工（含网格员）、国企员工、银行职员、年轻老师、展示馆讲解员、宣讲员、部队文职（辅导员 / 讲解员）。他们是最常提出写作、写稿、写材料需求的人群。

核心目标：

1. 用身份清晰、可搜索的标题和实用长文获取精准曝光。
2. 展示需求理解、结构设计、文字表达和交付能力。
3. 将有真实文稿需求的读者转化为咨询客户。
4. 沉淀可复用的选题、帖子模板、案例表达和运营经验。

## 发布形态

小红书与抖音统一发**长文图文**：一篇正文 + 一张封面图。不做视频。

## 项目结构

- `账号定位.md`：目标客户、内容方向、表达边界和转化路径。
- `内容规范.md`：标题、正文、封面、标签及发布前检查标准（双平台通用）。
- `选题池.md`：持续维护待创作选题和发布状态。
- `帖子模板.md`：与 AI 共创帖子时使用的输入、输出模板。
- `posts/`：归档已完成或已发布的帖子，一帖一文件。
- `assets/`：封面图、头像等静态素材。
- `案例汇总/`：脱敏前的案例原文素材（仅内部参考，不直接发布）。

## 后续共创方式

每次可以直接提供以下任意信息：

- 想推广的文稿类型；
- 一个客户需求或脱敏案例；
- 一段素材、截图或原始文稿；
- 想蹭的节点、话题或搜索关键词；
- 某篇帖子的实际数据和评论反馈。

默认输出：

1. 标题（可附 2—3 个备选）；
2. 可直接发布的正文；
3. 推荐话题标签；
4. 一张封面图；
5. 对应帖子归档文件（`posts/`）。

同一选题可双平台复用：小红书、抖音共用同一套标题、正文、标签和封面，按各平台字数限制微调即可。

## 基本原则

- 强调专业能力和解决问题，不制造身份焦虑。
- 案例必须脱敏，不上传、展示或复述涉密及敏感材料。
- 不冒充机关单位人员，不虚构客户评价和成交数据。
- 不承诺“包过”“内部渠道”等无法保证的结果。
- 避免生硬广告，以有用内容建立信任，再自然承接咨询。
