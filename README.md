# 文稿服务双平台运营项目

本项目用于持续运营一个面向职场人、国企事业单位从业者及其他泛体制内人群的专业内容账号，覆盖小红书与抖音。

核心目标：

1. 通过实用、专业、可信的文稿类内容获取精准曝光。
2. 展示需求理解、结构设计、文字表达和交付能力。
3. 将有真实文稿需求的读者转化为咨询客户。
4. 沉淀可复用的选题、帖子模板、案例表达和运营经验。

## 项目结构

- `账号定位.md`：目标客户、内容方向、表达边界和转化路径。
- `内容规范.md`：小红书标题、正文、标签及发布前检查标准。
- `抖音内容规范.md`：抖音视频结构、画面、字幕及发布前检查标准。
- `选题池.md`：持续维护待创作选题和发布状态。
- `帖子模板.md`：与 AI 共创帖子时使用的输入、输出模板。
- `视频脚本模板.md`：与 AI 共创抖音视频时使用的输入、输出模板。
- `posts/`：归档已完成或已发布的帖子，一帖一文件。
- `douyin/`：归档抖音脚本、发布文案和复盘数据，一条视频一文件。
- `output/douyin/`：存放可直接上传的抖音成片。

## 后续共创方式

每次可以直接提供以下任意信息：

- 想推广的文稿类型；
- 一个客户需求或脱敏案例；
- 一段素材、截图或原始文稿；
- 想蹭的节点、话题或搜索关键词；
- 某篇帖子的实际数据和评论反馈。

小红书默认输出：

1. 3—5个标题备选；
2. 一版可直接发布的正文；
3. 封面文案；
4. 推荐话题标签；
5. 评论区承接话术；
6. 对应帖子归档文件。

抖音默认输出：

1. 3秒开场钩子；
2. 45—60秒旁白和分镜；
3. 屏幕字幕与封面文案；
4. 发布文案、话题标签和置顶评论；
5. 对应视频归档文件；
6. 需要成片时，输出9:16无第三方水印视频。

## 基本原则

- 强调专业能力和解决问题，不制造身份焦虑。
- 案例必须脱敏，不上传、展示或复述涉密及敏感材料。
- 不冒充机关单位人员，不虚构客户评价和成交数据。
- 不承诺“包过”“内部渠道”等无法保证的结果。
- 避免生硬广告，以有用内容建立信任，再自然承接咨询。
